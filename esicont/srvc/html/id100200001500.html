การวิเคราะห์ปัญหาบนรถ

id100200001500


คุณสมบัติ

•  โมดูลที่เกี่ยวข้องกับระบบ CAN มีฟังก์ชั่นการวิเคราะห์ปัญหาบนรถ ฟังก์ชั่นการวิเคราะห์ปัญหาบนรถประกอบด้วยฟังก์ชั่นต่อไปนี้: ฟังก์ชั่นการตรวจจับความบกพร่อง ซึ่งจะตรวจจับความบกพร่องในส่วนที่เกี่ยวข้องกับระบบ CAN ฟังก์ชั่นหน่วยความจำ ซึ่งจะเก็บบันทึกรหัสวิเคราะห์ปัญหา ฟังก์ชั่นการแสดงผล ซึ่งจะแสดงบริเวณและสถานะที่บกพร่องผ่านจอแสดงรหัสวิเคราะห์ปัญหา

ฟังก์ชันการวิเคราะห์ปัญหาบนรถ


ฟังก์ชั่นการตรวจจับความบกพร่อง

•  ตรวจพบความผิดพลาดในสัญญาณอินพุทและเอาท์พุท
•  หากตรวจพบความบกพร่อง รหัสวิเคราะห์ปัญหาจะถูกส่งไปยัง DLC-2 ผ่านทางฟังก์ชันการแสดงความบกพร่อง ในขณะเดียวกัน ผลการตรวจจับความบกพร่องจะถูกส่งไปที่ฟังก์ชั่นหน่วยความจำ


ฟังก์ชั่นหน่วยความจำ

•  ฟังก์ชั่นหน่วยความจำจะเก็บบันทึกความบกพร่องในระบบสัญญาณอินพุท / เอาท์พุท ข้อมูลความบกพร่องจะไม่ถูกลบออกหลังจากบิดสวิตช์สตาร์ท OFF (LOCK) แม้ว่าระบบสัญญาณความบกพร่องได้กลับสู่ปกติแล้วก็ตามด้วยฟังก์ชั่นนี้
•  ข้อมูลความบกพร่องที่เก็บบันทึกไว้สามารถลบออกได้โดยการต่อ Mazda Modular Diagnostic System (M-MDS) เข้ากับ DLC-2 โดยปฏิบัติตามขั้นตอนในคู่มือซ่อมเพื่อลบรหัส


ฟังก์ชั่นการแสดงผล

•  ฟังก์ชั่นนี้ใช้สำหรับแสดงความบกพร่องปัจจุบันหรือความบกพร่องที่ผ่านมาผ่าน DLC-2 เป็นรหัสวิเคราะห์ปัญหา
•  การอ่านรหัสวิเคราะห์ปัญหาที่ปรากฏออกมาที่ DLC-2 ให้ใช้ Mazda Modular Diagnostic System (M-MDS)

ขั้นตอนการวิเคราะห์ปัญหา

•  หลังจากนำรถเข้ามา ให้ตรวจสอบใบสั่งซ่อมและอาการปัญหา และถ้าระบบ CAN เป็นสาเหตุของความบกพร่อง ให้ทำการวิเคราะห์ปัญหาระบบ CAN เพื่อตัดสินว่าความบกพร่องอยู่ในระบบ CAN หรือไม่
•  สำหรับการวิเคราะห์ปัญหาระบบ CAN ให้วัดแรงดันไฟฟ้าที่ขั้วเชื่อมต่อระบบ CAN ที่ DLC-2 สามารถตรวจสอบวงจรระบบ CAN หรือสามารถตัดสินอาการปัญหาได้ขึ้นอยู่กับค่าที่วัดได้
•  ถ้าอาการปัญหาไม่ใช่การขาดวงจร ให้ตรวจสอบแรงดันไฟฟ้าหรือความต่อเนื่องที่วงจรระบบ CAN และตัดสินชิ้นส่วนที่บกพร่อง
•  ถ้าอาการปัญหาเป็นการขาดวงจร ให้ตัดสินบริเวณที่มีการขาดวงจรโดยใช้รหัสวิเคราะห์ปัญหาของการสื่อสารผิดพลาดที่ปรากฏขึ้น และโมดูลซึ่งมีการสื่อสารล้มเหลว

ตัวอย่าง การตัดสินตำแหน่งการขาดวงจร
